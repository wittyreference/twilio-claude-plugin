# Segment Spec Reference

The Segment Spec defines six core API calls that all Segment libraries and the HTTP Tracking API implement. All calls share a set of common fields.

## Common Fields (All Call Types)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `anonymousId` | String | One of userId/anonymousId | Pseudo-unique substitute for userId; auto-generated by client SDKs |
| `userId` | String | One of userId/anonymousId | Unique database identifier for the user |
| `context` | Object | No | Contextual metadata (device, campaign, page, etc.) |
| `integrations` | Object | No | Enable/disable specific destinations. Default: all enabled. |
| `messageId` | String | Auto | Unique message ID (max 100 characters) |
| `timestamp` | Date | No | ISO-8601 UTC when the event occurred; defaults to server receipt time |
| `type` | String | Yes | One of: `identify`, `track`, `page`, `screen`, `group`, `alias` |

### Timestamp Semantics

| Timestamp | Source | Best For |
|-----------|--------|----------|
| `originalTimestamp` | Client clock at invocation | Used to calculate `timestamp`; unreliable alone |
| `sentAt` | Client clock at send | Clock skew correction |
| `receivedAt` | Segment server clock | Warehouse sorting; best when chronology doesn't matter |
| `timestamp` | `receivedAt - (sentAt - originalTimestamp)` | Downstream destinations; best when chronology matters |

All timestamps are ISO-8601 UTC. Engage requires timezone info in timestamps.

### Context Object

| Field | Type | Auto-collected | Description |
|-------|------|----------------|-------------|
| `active` | Boolean | No | User active flag; suppresses "last seen" updates |
| `app` | Object | Mobile | `name`, `version`, `build`, `namespace` |
| `campaign` | Object | Web (UTM) | `name`, `source`, `medium`, `term`, `content` |
| `device` | Object | Mobile | `id`, `advertisingId`, `adTrackingEnabled`, `manufacturer`, `model`, `type` |
| `ip` | String | Server-side | Filled by Segment servers for client events. IPv4 only. |
| `library` | Object | All | `name`, `version` |
| `locale` | String | All | e.g., `en-US` |
| `network` | Object | Mobile | `bluetooth`, `carrier`, `cellular`, `wifi` |
| `os` | Object | All | `name`, `version` |
| `page` | Object | Web | `path`, `referrer`, `search`, `title`, `url` |
| `screen` | Object | Mobile | `density`, `height`, `width` |
| `timezone` | String | All | tzdata format (e.g., `America/New_York`) |
| `traits` | Object | Prior Identify | User traits from most recent Identify call |
| `userAgent` | String | Web/Mobile | Device user agent string |
| `userAgentData` | Object | Web (Chrome) | Client Hints: `brands[]`, `mobile`, `platform` |
| `channel` | String | Auto | `server`, `browser`, or `mobile` |
| `groupId` | String | No | B2B group/account ID |

### Integrations Object (Destination Routing)

Opt-out model: all destinations enabled by default.

```json
{
  "integrations": {
    "All": true,
    "Salesforce": false,
    "Mixpanel": false
  }
}
```

Case-sensitive destination names. Must match the name in Segment documentation exactly.

## Identify

Ties a user to their actions and records traits. Call after registration, login, or profile updates.

```javascript
analytics.identify("97980cfea0067", {
  name: "Peter Gibbons",
  email: "peter@example.com",
  plan: "premium",
  logins: 5
});
```

### Reserved Traits

| Trait | Type | Description |
|-------|------|-------------|
| `address` | Object | `city`, `country`, `postalCode`, `state`, `street` |
| `age` | Number | User age |
| `avatar` | String | Avatar image URL |
| `birthday` | Date | User birthday |
| `company` | Object | `name`, `id`, `industry`, `employee_count`, `plan` |
| `createdAt` | Date | Account creation date (ISO-8601) |
| `email` | String | Email address |
| `firstName` | String | First name |
| `lastName` | String | Last name |
| `name` | String | Full name |
| `phone` | String | Phone number |
| `title` | String | Job title |
| `username` | String | Username (unique per user) |
| `website` | String | User website |

Reserved traits accept both camelCase and snake_case. Use database IDs (UUIDv4) for `userId`, not emails or usernames.

## Track

Records user actions with descriptive properties.

```javascript
analytics.track("Order Completed", {
  orderId: "50314b8e",
  total: 27.50,
  currency: "USD",
  products: [{ id: "507f1f77", name: "Widget", price: 18.99, quantity: 1 }]
});
```

### Reserved Properties

| Property | Type | Description |
|----------|------|-------------|
| `revenue` | Number | Dollar amount (decimal). Auto-converted for destinations like Mixpanel. |
| `currency` | String | ISO 4217 code. Defaults to USD if omitted. |
| `value` | Number | Abstract value for non-revenue events (e.g., newsletter signups) |

**Naming convention**: Human-readable noun + past-tense verb (e.g., "Order Completed", "Video Played"). Semantic event names have special mapping in downstream tools.

**Traits in Track**: Include `context.traits` to send user traits with a Track call to Actions destinations. This does NOT update Unify profiles — use Identify for that.

## Page

Records website page views. Analytics.js includes one `page()` call by default after `analytics.load()`.

```javascript
analytics.page("Docs", "Getting Started", {
  url: "https://example.com/docs/getting-started",
  referrer: "https://google.com"
});
```

### Reserved Properties

| Property | Type | Auto-collected | Description |
|----------|------|----------------|-------------|
| `path` | String | Yes | URL path (`location.pathname`) |
| `referrer` | String | Yes | Full previous URL (`document.referrer`) |
| `search` | String | Yes | Query string (`location.search`) |
| `title` | String | Yes | Page title (`document.title`) |
| `url` | String | Yes | Full URL; prefers canonical, falls back to `location.href` |
| `keywords` | Array[String] | No | SEO content keywords |

## Screen

Mobile equivalent of Page. Records screen views in mobile apps.

```swift
// Swift
analytics.screen(title: "Home", properties: ["variation": "blue signup button"])
```

```kotlin
// Kotlin
analytics.screen("Home", properties = buildJsonObject { put("variation", "blue signup button") })
```

Mobile SDKs can auto-capture screen transitions via `recordScreenViews()` / `trackApplicationLifecycleEvents`.

---

## Group

Associates users with organizations, companies, accounts, or teams. A single `userId` can belong to multiple `groupId` values.

```javascript
analytics.group("0e8c78ea9d97", {
  name: "Initech",
  industry: "Technology",
  employees: 329,
  plan: "enterprise"
});
```

### Reserved Group Traits

| Trait | Type | Description |
|-------|------|-------------|
| `address` | Object | `city`, `country`, `postalCode`, `state`, `street` |
| `avatar` | String | Group avatar URL |
| `createdAt` | Date | Creation date (ISO-8601) |
| `employees` | String | Employee count |
| `id` | String | Unique group DB ID |
| `industry` | String | Industry classification |
| `name` | String | Group name |
| `phone` | String | Phone number |
| `plan` | String | Service plan level |
| `website` | String | Website URL |

**No native group removal**: There is no API method to remove a user from a group. Device-mode: use Segment's `ready()` callback. Cloud-mode: use Destination Functions.

## Alias

Merges two unassociated user identities. Advanced method — only use when required for downstream destination compatibility.

```javascript
analytics.alias("new-user-id", "old-anonymous-id");
```

| Parameter | Required | Description |
|-----------|----------|-------------|
| `userId` | Yes | New identity to merge into |
| `previousId` | No | Existing ID (JS lib auto-passes anonymousId) |

**Alias does NOT merge profiles in Unify.** Use Identity Resolution for Unify profile merging.

For websites, call Alias client-side (anonymous ID is browser-generated). For server-side session IDs used as anonymous IDs, call from the server.

## Industry Specs

Segment provides recommended event schemas for specific verticals:

| Vertical | Key Events |
|----------|-----------|
| E-Commerce v2 | Product Viewed, Product Added, Order Completed, Checkout Started |
| Mobile | Application Installed, Application Opened, Application Updated |
| Video | Video Playback Started, Video Content Playing, Video Playback Completed |
| B2B SaaS | Account Created, Trial Started, Signed Up |
| AI Copilot | Copilot Opened, Message Sent, Suggestion Accepted |

Using industry-spec event names enables automatic mapping in destinations that support semantic events.
